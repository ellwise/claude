# Analyse with Swales

Map a text's communicative moves against its genre contract and classify each — with step-level detail — as Present, Stub, or Gap.

## Checklist

- [ ] Read the text completely before starting — skim nothing.
- [ ] **Genre** — "what communicative purpose does this text serve, for whom?" Identify the genre by purpose and discourse community, not by format or heading; if ambiguous, name the two most plausible candidates and ask the user to choose.
- [ ] **Move Inventory** — derive the move inventory for this genre and record each move as obligatory (omission is a failure), conventional (typical but contextually skippable), or optional.
- [ ] **Move Status** — produce a status table for all expected moves: *Present* (fully executed), *Stub* (nominally signalled but steps not executed — a heading or opening sentence alone qualifies), or *Gap* (entirely absent).
- [ ] **Step Coverage** — for each Present or Stub move, check whether all expected steps are executed; flag underrealized steps (a step that exists but is incomplete).
- [ ] **Move Ordering** — flag inversions where a move appears before the preceding moves have prepared the reader for it.
- [ ] **Genre Bleed** — flag content that belongs to a different genre.
- [ ] Before reporting: review each finding for conflicting forces (would fixing this create a different problem?) and significance (is this too marginal to act on?). Aim to converge — discard advice that churns rather than converges.
- [ ] **Summary** — report in plain language, ordered by severity: obligatory gaps (name the harmed reader and specific steps to fill the gap) → stubs and underrealized steps (with steps to complete them) → ordering and bleed issues → optional moves worth adding. Offer to dig deeper on any finding.
